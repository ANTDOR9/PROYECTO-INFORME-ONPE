01 · Introducción Ejecutiva

El sistema que necesita
la ONPE.

Tras las fallas del STAE en la primera vuelta del 12 de abril, la ONPE necesita un sistema rediseñado y robusto para procesar 86,488 mesas en tiempo real.

● El Problema

El sistema STAE tuvo fallas críticas durante la primera vuelta, extendiendo las votaciones hasta el lunes 13 de abril. La ONPE decidió eliminarlo y rediseñar desde cero.

● El Encargo

Modelar el SED-2026: el sistema interno que recibe actas de mesas, valida los datos, los suma al total nacional y publica resultados oficiales en tiempo real.

● La Escala

  • 27 millones de electores habilitados
  • 86,488 mesas de sufragio
  • 126 centros de cómputo descentralizados
  • 900,000 votos en el exterior

03.1 · Especificación Técnica

CU: Registrar Acta de Mesa

Actor PrincipalPresidente de Mesa
Actores SecundariosRENIEC, Servicio SMS, JNE
PrecondicionesSesión iniciada; Mesa en estado "Cierre de Escrutinio".

● Flujo Principal

  1. El sistema solicita «include» Autenticar Presidente de Mesa.
  2. RENIEC valida la identidad del actor.
  3. El sistema despliega el formulario de entrada de resultados.
  4. Se invoca «include» Validar Suma de Votos para consistencia aritmética.
  5. El sistema registra el acta firmada digitalmente.
  6. Se activa «include» Enviar Notificación SMS.

● Flujos de Extensión

  • Si intentos fallidos > 3: «extend» Bloquear DNI.
  • Si suma no cuadra: «extend» Marcar Acta como Observada.
  • Notificación JNE: El JNE recibe alerta automática en caso de observación.

09 · Reflexión Final

¿Qué decisión es más crítica?

El STAE falló. ¿Qué decisión arquitectónica en el SED-2026 evita que eso vuelva a ocurrir?

"El STAE falló porque era un único componente monolítico. La decisión más crítica en el SED-2026 es la API Central como hub distribuidor de responsabilidades — elimina el punto único de fallo que tumbó las elecciones de abril."

● El error del STAE

Arquitectura monolítica con punto único de fallo (SPOF). Al fallar un solo componente, todo el proceso electoral colapsó. 27 millones de electores dependían de un sistema sin redundancia.

● La solución SED-2026

La API Central distribuye la carga entre módulos independientes. Si el Módulo de Notificaciones falla, el registro continúa. Cluster EC2 de 3 instancias + PostgreSQL Multi-AZ eliminan el SPOF.

● Decisiones clave

  • API Central como hub único de entrada
  • Módulos especializados e independientes
  • Cluster de 3 instancias EC2
  • BD PostgreSQL Multi-AZ
  • Paralelismo Regla #4 (fork-join)
